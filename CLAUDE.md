## הכיוון: עשיר ודאטה-מוכוון (לא מינימלי)

המערכת צריכה להרגיש כמו פלטפורמת אנליטיקס מלאה — **צ'ארטים, צבע, וצפיפות** — לא טבלה מינימלית.
כל מסך אנליטי מורכב משלוש שכבות (חלק ו', סעיף 26):
1. שורת KPI צבעונית (4 כרטיסים, כל אחד עם צבע + בר + טרנד) — סעיף 28.
2. פאנלים של צ'ארטים (גרף מגמה גדול + דונאט) — סעיף 29.
3. טבלת נתונים מפורטת — סעיף 11.
אם נשאר ואקום במסך — להוסיף פאנל/צ'ארט, לא אוויר.

## כללי ברזל (תקציר — הפירוט המלא במסמך)

- **צבעים:** רק CSS variables. אסור hardcoded hex. פלטת הצ'ארטים בסעיף 27 (עד 5 צבעים, כל מטריקה בצבע קבוע: spend=רוז-גולד, leads=ירוק, impressions=כחול).
- **פונטים:** מספרים ← `JetBrains Mono` (כולל צירי צ'ארטים). כותרות/הירו ← `Fraunces`. UI ← `Inter`. רק משקלים 400/500.
- **צ'ארטים:** Chart.js 4.x מ-cdnjs. קו tension 0.4, מילוי גרדיאנט, נקודות רק ב-hover, grid עדין. דונאט cutout 68% עם טקסט מרכזי. (סעיף 29)
- **החלפת theme:** לבנות צ'ארטים מחדש (destroy+recreate) כי הצבעים מ-CSS variables. (סעיף 31)
- **עומק:** לכל כרטיס/פאנל צל (--shadow) שמפריד מהרקע. לא שטוח.
- **רוז-גולד:** צבע המותג. נשאר על המספר/פעולה הראשית ועל נתון-המפתח. צבעי הצ'ארט נפרדים ממנו.
- **שני מצבים:** כל צבע עובד ב-dark וב-light. הסמנטיים מכהים ב-light.
- **טופבר עשיר:** חיפוש + מייל + פעמון + אווטאר. (סעיף 30)
- **Sentence case** בטקסטים.
