## הפרומט — להדביק ל-Claude Code

```
קרא את docs/design-system.md (מקור האמת) ואת docs/nextadsgen-dashboard-v4.html
(מימוש הרפרנס החי). אנחנו מיישמים את הכיוון העשיר והדאטה-מוכוון על המערכת החיה.

הקונטקסט: המערכת הנוכחית מינימלית מדי — טבלאות ומספרים בלי ויזואליזציה.
היעד: שתיראה כמו פלטפורמת אנליטיקס מלאה — צ'ארטים, KPI צבעוניים, צפיפות, עומק.
הרפרנס v4 הוא בדיוק איך זה צריך להיראות.

המשימה (התחל ממסך Campaigns בלבד — שאר המסכים בהמשך):
1. הגדר את כל ה-CSS variables מסעיף 2 + הפלטה המורחבת מסעיף 27 (root + [data-theme]).
   ודא ש-dark ו-light שניהם מוגדרים במלואם.
2. החלף את שורת ה-KPI הקיימת בכרטיסי KPI צבעוניים לפי סעיף 28:
   פס צבע אנכי, אייקון צבעוני, בר התקדמות, וטרנד. כל מטריקה בצבע הקבוע שלה
   (spend=accent, leads=green, CPL=blue, qualified=amber).
3. הוסף שני פאנלים מעל הטבלה (סעיף 29):
   - פאנל רחב: גרף מגמה (Chart.js line) של spend ו-leads ל-30 יום, מילוי גרדיאנט,
     עם שורת סטטים מעל (total spend / leads / impressions), כל סטט בצבע הסדרה.
   - פאנל צר: דונאט (Chart.js) של spend לפי קמפיין, עם סכום כולל במרכז ו-legend.
4. הוסף טופבר עשיר לפי סעיף 30: חיפוש + מייל + פעמון + אווטאר.
5. ודא שהצ'ארטים נבנים מחדש בהחלפת theme (destroy+recreate, סעיף 31).

חוקים מחייבים:
- כל צבע = CSS variable. אסור hardcoded hex.
- כל מספר → JetBrains Mono (כולל צירי הצ'ארטים). כותרות/הירו → Fraunces. UI → Inter.
- רק משקלים 400/500.
- לכל כרטיס/פאנל צל (--shadow) שמפריד מהרקע.
- Chart.js 4.x מ-cdnjs. צבעי הצ'ארט נקראים מ-CSS variables ב-runtime, לא קבועים בקוד.

השתמש ב-data האמיתי שכבר קיים במערכת (לא נתוני דמה) — הרפרנס v4 משתמש בנתוני
דוגמה, אבל אתה צריך לחבר לנתונים החיים של המשתמש. אם אין עדיין time-series לגרף
המגמה (נתונים יומיים), השתמש בנתונים שיש ובנה את ה-aggregation, או שאל אותי.

בסיום, עבור על הצ'קליסט בסעיף 23 + סעיף 32 בשני המצבים (dark + light).
אל תמציא ערכים שלא במסמך. אם חסר משהו — עצור ושאל.
```

## טיפים

**עבוד מסך-מסך, לא הכל בבת אחת.** התחל מ-Campaigns (יש לו רפרנס מלא). אחרי שהוא נראה
כמו ה-v4, עבור ל-Dashboard, ואז Intelligence, וכך הלאה. כל פעם הפרומט זהה — רק החלף
את שם המסך.

**אם משהו יוצא שונה מהרפרנס** — אל תתקן ידנית. אמור ל-Claude Code:
"השווה את התוצאה ל-docs/nextadsgen-dashboard-v4.html — [X] לא תואם. תקן לפי הרפרנס."

**אחרי כל מסך** — בדוק dark וגם light, ובדוק שהצ'ארטים לא נשברים בהחלפה.

**הנתונים** — הרפרנס משתמש בסדרות דמה לגרף. במערכת החיה תצטרך endpoint שמחזיר
spend/leads יומי ל-30 יום. אם אין — בקש מ-Claude Code לבנות את ה-aggregation מהנתונים
הקיימים, או להתחיל עם הנתונים שיש (sparkline פשוט) ולשדרג אחר כך.
